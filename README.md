## 📝 1. 프로젝트 개요
**“우리는 모두 언젠가 백수가 된다.”**
<br>

많은 청년들이 무업 기간을 겪으며 불안, 단절, 우울을 경험합니다. 니트생활자는 무업 기간 사회적 단절을 경험하는 청년들이 연결되는 커뮤니티 플랫폼입니다. **력습지**는 이런 무업 청년들의 일상에 찾아오는 무기력 및 우울 관리를 위한 학습지 서비스입니다. 청년들이 무기력의 늪에 빠지지 않도록 경험을 통한 성취감을 고취하고 사회적인 연결을 만드는 활동입니다. 첫 번째로는 할 일을 부여해 소소한 성취감부터 시작해서 다른 것들도 해낼 수 있는 마음상태를 만들고, 두 번째로는 같은 상황에 놓인 청년들 간의 커뮤니티를 형성해 공존을 위한 연결을 만들고, 세 번째로는 시간이 지남에따라 성장하는 과정을 시각화해 무업 기간을 전환 기간으로 바꾸는 것이 목표입니다.

기존에도 수기로 력습지를 진행했으나, 수기로 진행했기 때문에 <font color="red">**인력 의존도가 높다는 단점**</font>이 있었습니다. 또한, 대화와 미션 완수 모두 카카오톡 채팅방에서 일어나서 사용에 불편함이 있었습니다. 따라서 본 프로젝트는 **기존 력습지 서비스를 자동화**하여 관리할 수 있는 관리자 웹페이지, 그리고 미션 수행 및 성취를 한눈에 볼 수 있는 사용자 앱으로 나누어서 진행하였습니다.

특히, 이전 력습지 관리자들은 사용자 개인마다 미션 완료 유무를 체크하고, 이에 대해 리마인드를 주기적으로 시행해야 했기에 쓸데 없이 많은 시간을 할애했습니다. 이러한 불편함과 시간 낭비를 줄이기 위해, 웹 형식의 관리자 사이트를 만들었으며, 많은 자료들을 한 눈에 확인 가능하게끔 제작되었습니다. 


## 💻 2. 프로젝트 설명

관리자 페이지에서는 팀별로 속해있는 사용자들의 미션 완료 현황을 볼 수 있으며, 새롭게 사용자에게 미션 할당이 가능합니다. 또한, 팀과 사용자의 삭제/추가/수정이 가능한 페이지를 추가로 제작하였습니다. 해당 기능을 구현한 4개의 페이지로 구성되어 있습니다.

### A. 통계 탭
---
통계 탭은 사이트의 첫 화면으로 보여지는 탭이며, 사용자들의 미션 달성 현황을 한 눈에 볼 수 있습니다. 

<br>

![20231201_020834](https://github.com/tylerkim1/RYUK_Manage/assets/125373555/f59ea4ef-1dec-4fa2-a8b0-4ff94fcd1f90)


위의 화면처럼 **기간 범위, 사용자 범위, 날짜**를 선택하면 해당 선택에 맞는 미션 성취율을 보여줍니다. 

<u>**미션 성취율**</u>이란, 실행해야 하는 전체 미션 개수에 대한 완료된 미션 개수의 퍼센트를 의미합니다. 성취율를 시각적으로 보여주기 위해, 성취율에 비례하여 <font color="red">빨간 칸</font>의 길이가 늘어납니다. 데이터는 성취율에 대한 내림차순으로 정렬됩니다.

또한, 데이터 위에 있는 3가지 설정 칸을 사용하여, 원하는 데이터를 얻을 수 있습니다.

<br>

#### ⏱기간 범위
첫번째 선택 칸은 기간 범위를 설정하는 칸입니다. 이는 드롭다운 형태로 구현되며, 총 2가지 선택지가 존재합니다.
- **일별**: 하루 치의 통계 자료를 보여줍니다.

- **월별**: 한달 치의 통계 자료를 보여줍니다.

<br>
  
#### 🙍‍♂️사용자 범위
두번째 선택 칸은 사용자 범위를 설정하는 칸입니다. 이 또한, 2개의 선택지를 가진 드롭다운 형태입니다. 
- **개인별**: 선택 기간에 참여 중인 모든 사용자들의 통계 자료를 각각 보여줍니다.

- **팀별**: 선택 기간에 존재하는 모든 팀별 통계 자료를 각각 보여줍니다. 팀별 통계 자료란, 팀 내에 속해있는 모든 사용자들의 미션 개수를 통합한 데이터를 의미합니다. 

아래 화면은 팀별을 선택했을 시 보여지는 화면 예시입니다.

<br>

![20231201_021604](https://github.com/tylerkim1/RYUK_Manage/assets/125373555/138b1788-aa8e-4b58-a03e-ac0500809433)


<br>

#### 📅날짜
마지막으로, 관리자는 원하는 날짜를 선택하여 통계를 볼 수 있습니다. 기간 범위가 월별일 경우, 원하는 달을 선택할 수 있으며, 일별일 경우, 원하는 날을 선택할 수 있습니다.

하지만, 해당 팀 혹은 개인이 력습지에 가입되기 전 날짜는 선택이 불가능합니다. 현재 시점 이후의 날짜 또한 고를 수 없습니다.

### B. 미션 탭
---
미션 탭은 각 미션별 진행률과 회원들에게 새로운 미션을 할당할 때 사용됩니다.

<br>


#### 👀 미션 보기
사이드 바 두번째 ***미션***을 누를 시, 처음으로 열리는 페이지는 특정 팀에 할당된 미션을 보여주는 창입니다. 각각의 미션에 대해 제목, 카테고리, 성취율 순으로 나타납니다. 
![20231201_021512](https://github.com/tylerkim1/RYUK_Manage/assets/125373555/010b0c93-ae73-4049-a5f0-a9eb7ec7bf04)

여기서도 동일하게 3가지 설정을 자유롭게 선택하여 원하는 데이터를 얻을 수 있습니다.

- **팀**: 현재 존재하는 팀 중 하나를 선택할 수 있습니다.

- **카테고리**: "전체", "매일하력", "시도해력", "마음봄력", "유유자력", "레벨업력" 중 하나를 선택하여 해당 카테고리에 속한 미션들만 보이게 설정합니다.

- **날짜**: 선택한 팀의 활동 기간 중 하루를 선택하여 해당 일에 팀에게 할당된 미션을 확인할 수 있습니다.(미래 날짜는 선택이 불가능합니다)


<br>

#### 📣미션 할당
우측 상단에 있는 ***미션 할당*** 버튼을 누를 시, <u>이미 미션 풀에 존재하는 미션</u> 중 하나를 특정 팀에게 추가할 수 있습니다. 

<br>

</p>

<br>

#### ⚙미션 풀 관리
위에서 언급한대로, 미션 풀 내에 존재하는 미션들만 팀에게 추가할 수 있습니다. 미션 풀을 관리하기 위해, 미션 할당 탭의 우측 상단에 ***미션 풀 관리*** 버튼이 존재합니다. 

여기서는 기본적으로 미션 풀에 어떤 미션이 들어있는지 한 눈에 확인이 가능합니다. 미션의 제목 뿐 아니라, 미션들의 카테고리까지 보여줍니다. 그리고, 미션 풀에 미션을 추가하거나, 삭제하는 기능 역시 미션 풀 관리에서 실행할 수 있습니다. 삭제는 간단히, 미션 옆 X자를 클릭하면 됩니다. 추가는 다음 **새로운 미션 추가**부분에 자세히 나와있습니다. 

<br>

<p align = "center">
<img src = "https://github.com/tylerkim1/RYUK_Manage/assets/125373555/c6845838-87b8-4469-a0d5-01a01525ec87" width="300">
</p>

<br>

#### 🔥새로운 미션 추가
미션 풀에 존재하지 않는 미션을 추가하고 싶다면, 미션 풀에 먼저 추가한 후, 팀에게 할당해야 합니다. 

이를 위해, 미션 풀 관리 탭의 우측 상단에 ***새로운 미션 추가*** 버튼이 존재합니다. 이 버튼을 누르면 다음의 화면이 뜨면서 미션 풀에 새로운 미션을 추가할 수 있습니다.

<br>

<p align="center">
<img src = "https://github.com/tylerkim1/RYUK_Manage/assets/125373555/12c2b504-2519-4a66-bbf4-b05a8af47337" width = "300">
</p>

미션의 제목을 입력하고 카테고리를 선택한 후 ***미션 추가하기*** 버튼을 눌러 제출하면, 미션 풀에 해당 미션이 새롭게 추가됩니다.

<br>

### C. 팀관리 탭
---

팀관리 탭은 현재 어떤 팀이 존재하며, 팀 내의 회원들을 관리하기 위한 탭입니다.

<br>

#### 👀전체 팀 보기 & 검색
사이드 바 세번째 ***팀관리*** 를 누를 시, 처음으로 열리는 페이지는 현재 시점 존재하는 팀들을 보여주는 창입니다.

다음처럼, 전체 팀에 대한 팀의 **이름**과 해당 팀의 **소개글**, **회원 명수**를 보여주고 있습니다.
<img src = "https://github.com/tylerkim1/RYUK_Manage/assets/125373555/aa39b92f-3f8b-4897-b092-12d40f9cab47" >

좌측 상단에는 팀명에 대한 **검색창**도 존재합니다. 팀의 개수가 많아, 특정 팀을 찾기 힘들 때, 검색을 통해 쉽게 팀을 찾을 수 있습니다.

<br>

#### 🧐특정 팀 보기
특정 팀을 누르면, 해당 팀에 속한 회원들의 이름을 전부 보여줍니다. 만약, 해당 팀을 아예 삭제하고 싶다면, 우측 상단에 있는 ***삭제*** 버튼을 눌러 팀을 삭제합니다. 또한, 팀이 삭제되면 팀에 배정된 모든 회원들은 해당 팀에서 자동으로 나가게 됩니다.

<br>

<p align="center">
<img src = "https://github.com/tylerkim1/RYUK_Manage/assets/125373555/20312e6b-38c6-437a-8cd7-8bdc921a7deb"  width="300">
</p>

<br>


#### 🔥팀 추가
팀 관리 탭에서 우측 상단에 있는 ***팀 추가*** 버튼을 누르면, 새로운 팀을 만들 수 있습니다. 

팀을 새로 제작할 시에는 필요한 정보는 다음과 같습니다.
- **팀이름**: 기존에 있는 팀 이름과 중복되지 않는 이름으로 선정합니다

- **링크**: 팀원들끼리 소통할 오픈 카카오톡 채팅 링크

- **팀장**: 팀장을 맡을 회원의 ID를 작성

- **팀 분야**: 해당 팀의 결성 목적 ex) 운동, 독서

-  **소개글**: 해당 팀에 대한 간략한 소개

- **기간**: 해당 팀의 활동 시작 날짜와 종료 날짜 

<br>

<p align="center"><img src ="https://github.com/tylerkim1/RYUK_Manage/assets/125373555/c6e0f441-8172-48c7-860b-aa04a5599b25" width = "300">
</p>
팀을 만든 직 후에는, 팀장을 제외한 어떠한 팀원도 팀에 속하지 않습니다. 이 후, 회원관리 탭 혹은 사용자 앱을 사용하여 팀에 회원들을 추가할 수 있습니다.

<br>

### D. 회원관리 탭
------
회원관리 탭은 관리자와 사용자의 모든 정보를 보거나, 수정/삭제하기 위해 사용됩니다.  해당 페이지는 크게 3가지 구성요소로 나누어 설명할 수 있습니다.

아래 화면은 사이드 바 네번째 ***회원관리*** 버튼을 눌렀을 때 처음 뜨는 창으로, 스크롤 바를 내리면 사용자 목록 아래로 관리자 목록도 나타납니다.
<img src="https://github.com/tylerkim1/RYUK_Manage/assets/125373555/bce3b98b-4e3f-4e78-a344-665dd99a1cd1">

<br>

#### 🔥회원 추가
페이지의 가장 위에 존재하는 회원 추가는 **사용자 혹은 관리자를 추가**하기 위해 사용됩니다. 처음 회원을 만들 때에는 팀 설정이 불가하며 어떠한 팀에도 속하지 않는 상태로 존재합니다. 또한, 회원을 추가한 후 회원의 식별 id(로그인 ID와 다름)가  랜덤으로 생성됩니다. 이 id는 회원들을 식별하는 데 사용되기에 회원끼리 중복되지 않으며, 목록에서 "#"아래에 적혀진 숫자가 id를 나타냅니다.

회원을 추가할 때 필요한 정보로는 "이름", "닉네임", "ID", "관리자 유무", "이메일", "비밀번호"가 있습니다. 다른 정보들은 회원끼리 중복이 가능하나, ID가 중복되서는 안됩니다.
<img src="https://github.com/tylerkim1/RYUK_Manage/assets/125373555/ac8b75ed-d8b2-4038-ba28-895fafb11b63" >

좌측 상단에 ***추가*** 버튼을 누를 시, 해당 정보를 가진 회원이 추가됩니다. 


<br>

#### 🙌사용자 목록
추가 탭 아래에는 사용자들의 정보를 담은 사용자 목록이 보여집니다. 보여지는 정보로는 사용자들의 **식별 ID, 이름, 로그인 ID, 닉네임, 이메일, 속한 팀**이 있습니다. 비밀번호는 보안 차원에서 보여지지 않으며, 회원이 처음 생성된 후 팀에 할당하지 않으면 회원의 팀 정보는 빈칸으로 보입니다.

해당 목록은 이름의 내림차순 순서대로 정렬되며, 팀관리 탭과 동일하게 사용자 이름을 이용하여 원하는 사용자를 검색할 수 있습니다.

또한, 회원 정보 우측에 존재하는 버튼을 사용하여 회원의 정보를 수정하거나, 회원을 삭제할 수 있습니다. 삭제는 단순히 버튼을 누를 시 실행됩니다. 수정은 **회원 정보 수정**에서 자세히 설명할 것입니다.

#### ✍관리자 목록
사용자 목록과 거의 유사한 형태입니다. 다만, 관리자는 팀에 소속되지 않은 회원이기에, 팀 정보가 보여지지 않습니다.

#### ✒회원 정보 수정
사용자와 관리자 둘 다, 관리자측에서 자유롭게 정보를 수정할 수 있습니다. 이는 ***수정*** 버튼을 눌러 나오는 팝업 창을 사용하여 실행할 수 있습니다.
<br>

<img src=" https://github.com/tylerkim1/RYUK_Manage/assets/125373555/776b9fc3-5f4e-481e-ba85-d7572c2680c7" width = "300"> <img src="https://github.com/tylerkim1/RYUK_Manage/assets/125373555/e651ab5d-ca5e-421e-825e-d17dd39d305d" width="326">

사용자와 관리자의 이름, 닉네임, 로그인 ID, 이메일, 비밀번호를 수정할 수 있으며, 사용자의 경우 속한 팀도 현재 존재하는 팀 중 하나를 선택하여 할당하거나, 변경할 수 있습니다. ***수정*** 버튼을 누르면, 기존의 정보에서 수정된 정보로 값이 변경됩니다. (로그인 ID 수정 시, 중복되지 않는 ID여야 합니다.)

### E. 탈퇴 요청
---
사용자가 자신이 속한 팀에서 탈퇴하고 싶을 때에는, 이를 관리자가 승인해야 합니다. 요청 확인과 승인을 위해 사이드 바 아래에 **탈퇴 요청** 칸이 존재합니다.


<br>

<img src="https://github.com/tylerkim1/RYUK_Manage/assets/125373555/d926a2e0-dd76-435d-bfe0-e92c15eeb2bf" >

어떤 사용자 한 명이 팀에서 탈퇴를 누르면, 위의 사진처럼 탈퇴 요청 목록에 사람이 한 명 추가되며, 팝업창에는 탈퇴를 누른 사용자의 이름과 탈퇴할 팀의 이름이 보여집니다. 여기서 ***탈퇴*** 버튼을 누를 시, 해당 사용자는 속한 팀에서 나가게 됩니다. 탈퇴를 원하는 사람이 여러명의 경우, 탈퇴 요청 목록에 여러 개의 요청이 뜹니다. **탈퇴 요청** 칸 옆에 뜨는 숫자는 탈퇴를 요청한 사용자의 수를 의미합니다.

만약에 요청을 허가하지 않는다면, 좌측에 "X"버튼을 누르면 됩니다.

## 3. 실행방법
현재는 서버에 올리지 않은 상태입니다. 그럼으로 다음의 방식을 따라 local 환경에서 실행이 가능합니다.
##### 1. github의 web URL을 "git clone"하여 해당 깃헙을 local에 다운받는다.
##### 2. 터미널에 다음 순서대로 입력한다.

    npm install
    npm start

(사용가능한 데모 링크 (ex. 데모 웹사이트 링크, 실행가능한 Google colab 파일, 데모 앱 설치 링크 …)
빌드 / 설치 / 실행 방법 및 환경 설정에 대한 설명)
